# AI 辅助编程完整流程指南
## 一、初始需求设想
用户初始提出的 AI 编程基本步骤：
1. 获取需求与在 chat 模式下整理出设计方案（包括要满足什么功能、使用什么技术栈、要满足什么风格等），最好整理为 readm.md markdown 文本；
2. 按照设计方案进行编码；
3. 测试功能，排除 bug（包括风格调试，细节微调）；
4. 打包环境，编写 shell 脚本。

## 二、AI 辅助编程标准化流程
本文档基于初始需求，梳理了使用 AI 辅助完成编程开发的标准化步骤，涵盖从需求分析到最终交付的全流程，帮助高效、规范地完成开发任务。

### 第一步：需求分析与方案设计
#### 核心目标
明确开发边界、技术选型和交付标准，形成可落地的设计方案（建议整理为 `README.md`）。

#### 具体操作
1. **需求拆解**
   - 与产品/需求方确认核心功能（必须实现）、次要功能（可选实现）、非功能需求（性能、兼容性、安全性等）；
   - 用 AI（如 ChatGPT/文心一言）以对话模式梳理需求，输出结构化的需求清单；
   - 示例：
     ```markdown
     # 需求清单
     ## 核心功能
     1. 用户登录/注册（支持手机号+验证码）
     2. 数据列表展示（支持分页、筛选）
     ## 非功能需求
     1. 页面加载时间 < 2s
     2. 兼容 Chrome/Firefox 主流版本
     ```

2. **方案设计**
   - 技术栈选型（结合 AI 建议+项目实际）：
     - 前端：React/Vue + TypeScript + Ant Design；
     - 后端：Python/Java + FastAPI/SpringBoot；
     - 数据库：MySQL/Redis/MongoDB；
   - 风格规范：UI 风格（简约/商务/科技）、代码规范（ESLint/PEP8）、命名规范等；
   - 输出 `README.md` 设计方案，包含：
     - 项目名称与简介；
     - 功能清单（分优先级）；
     - 技术栈明细；
     - 界面风格参考；
     - 交付物清单。

#### AI 辅助技巧
向 AI 提问模板：`帮我梳理【XX项目】的开发方案，核心需求是XX，要求兼容XX环境，优先使用XX技术栈，输出结构化的README.md格式`

### 第二步：按方案编码实现
#### 核心目标
基于设计方案，分模块完成代码编写，保证代码可运行、符合规范。

#### 具体操作
1. **模块拆分**
   - 将整体功能拆分为独立模块（如用户模块、数据模块、UI 组件模块）；
   - 按模块依次向 AI 提需求，避免一次性生成大量代码导致混乱。

2. **AI 辅助编码**
   - 针对单个模块，向 AI 提供明确指令：
     - 示例：`帮我用 Vue3 + Element Plus 实现用户登录组件，包含表单验证、记住密码功能，代码符合ESLint规范`；
   - 逐段验证 AI 生成的代码：
     - 检查语法错误、逻辑漏洞；
     - 确保代码与技术栈版本兼容；
     - 补充必要的注释（函数用途、参数说明、特殊逻辑）。

3. **代码整合**
   - 将各模块代码整合到项目目录中，保证目录结构清晰：
     ```
     project/
     ├── src/          # 源码目录
     │   ├── components/  # 组件
     │   ├── api/         # 接口请求
     │   └── utils/       # 工具函数
     └── README.md     # 设计方案
     ```

#### 注意事项
- 不直接照搬 AI 生成的代码，需结合业务场景调整；
- 关键逻辑（如权限、支付）需人工复核，避免 AI 生成的漏洞。

### 第三步：功能测试与细节优化
#### 核心目标
验证功能完整性，修复 Bug，优化体验细节。

#### 具体操作
1. **功能测试**
   - 按需求清单逐条验证功能：
     - 核心功能是否全部实现；
     - 边界场景（空输入、异常请求）是否处理；
     - 数据交互（接口请求、数据库读写）是否正常；
   - 借助 AI 生成测试用例：
     - 示例：`帮我生成【用户登录模块】的测试用例，包含正常场景、异常场景`。

2. **Bug 排查与修复**
   - 记录测试中发现的 Bug（现象、复现步骤）；
   - 向 AI 描述 Bug 现象，获取排查思路：
     - 示例：`Vue3 组件中点击登录按钮无响应，控制台无报错，帮我分析可能原因并给出修复方案`；
   - 修复后重新测试，确认 Bug 闭环。

3. **细节优化**
   - 风格调试：UI 样式（颜色、间距、字体）与设计方案对齐；
   - 交互优化：按钮点击反馈、加载状态提示、错误提示文案；
   - 性能微调：优化重复渲染、减少无效接口请求。

#### 常用工具
- 前端：Chrome DevTools、Vue DevTools；
- 后端：Postman（接口测试）、日志工具；
- 通用：AI 代码审查（如 `帮我审查这段Python代码的潜在问题`）。

### 第四步：环境打包与脚本编写
#### 核心目标
打包可交付的运行环境，编写自动化脚本提升部署效率。

#### 具体操作
1. **环境打包**
   - 前端项目：
     ```bash
     # 示例：Vue 项目打包
     npm run build  # 生成 dist 目录
     ```
   - 后端项目：
     - Python：打包为 `requirements.txt` 或 wheel 包；
     - Java：打包为 Jar/War 包；
   - 整理环境依赖清单：
     ```markdown
     # 环境依赖
     - Python 3.9+
     - Redis 6.0+
     - 依赖包：requirements.txt
     ```

2. **Shell 脚本编写**
   - 核心脚本类型：
     1. 环境部署脚本（`deploy.sh`）：自动安装依赖、配置环境；
     2. 启动脚本（`start.sh`）：一键启动项目；
     3. 停止脚本（`stop.sh`）：安全停止服务；
   - 脚本示例（启动前端服务）：
     ```bash
     #!/bin/bash
     # start.sh - 启动前端静态服务
     PORT=8080
     # 检查端口是否被占用
     if lsof -i:$PORT; then
         echo "端口 $PORT 已被占用，正在关闭占用进程..."
         lsof -t -i:$PORT | xargs kill -9
     fi
     # 启动服务
     echo "启动前端服务，端口：$PORT"
     cd dist && python -m http.server $PORT
     ```

3. **交付物整理**
   - 打包内容：编译后的代码、环境依赖清单、Shell 脚本、说明文档；
   - 编写部署说明（`DEPLOY.md`）：脚本使用方法、环境要求、常见问题。

#### 注意事项
- 脚本添加注释，便于后续维护；
- 测试脚本的兼容性（不同 Linux 发行版）；
- 敏感信息（数据库密码）通过环境变量传入，避免硬编码。

## 三、核心要点总结
1. **方案先行**：需求分析和方案设计是基础，需形成结构化的 `README.md`，明确技术栈和功能边界；
2. **编码可控**：AI 生成的代码需人工复核调整，按模块拆分开发更高效；
3. **交付完整**：不仅要实现功能，还需打包环境、编写自动化脚本，降低部署成本。
